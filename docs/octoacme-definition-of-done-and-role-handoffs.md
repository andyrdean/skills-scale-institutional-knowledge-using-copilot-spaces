# OctoAcme — Definition of Done & Role Handoffs

## Purpose
Provide a reusable checklist that makes Definition of Done decisions and cross-role hand-offs explicit from planning through release and post-release support.

## Checklist
- [ ] Scope and acceptance criteria are finalized and linked to the work item. (Owner: PdM)
- [ ] Delivery sequencing, dependencies, and milestone timing are confirmed. (Owner: PM)
- [ ] Implementation is complete and code review feedback is resolved. (Owner: Developers)
- [ ] Unit/integration coverage is updated for changed logic. (Owner: Developers)
- [ ] UX/UI behavior and accessibility expectations are validated. (Owner: UX/UI Designer)
- [ ] Feature acceptance and regression validation are complete. (Owner: QA/Test Lead)
- [ ] Security checks/scans pass, and high-risk findings have owners. (Owner: Security Champion)
- [ ] Release notes and deployment/rollback plan are ready. (Owner: Release Manager)
- [ ] Support runbook/known issues and escalation contacts are updated. (Owner: Support/Success Engineer)
- [ ] Final go/no-go decision is recorded with owners and date. (Owner: PM + Release Manager + PdM)

## How to use
- Use during planning to align on acceptance criteria and hand-off expectations before implementation begins.
- Re-check during execution when moving work into QA and Done in the board workflow.
- Use again during release readiness as a final cross-functional sign-off before production deployment.
- Pair with [OctoAcme — Execution & Tracking](./octoacme-execution-and-tracking.md) and [OctoAcme — Release & Deployment Guide](./octoacme-release-and-deployment.md).
